# IDEA 安装

使用的TRAE工具或者VSCode

## 安装solidity相关插件
1. Code Spell Checker (streetsidesoftware.code-spell-checker)
作用：通用拼写检查器，支持 Solidity 文件里的英文注释、变量名、字符串等。
特点：不是 Solidity 专属，而是全语言通用。会标记拼写错误（如 "contrcat" 变红），可自定义词典。
和 Solidity 关系：辅助工具，帮助写注释/文档时避免低级拼写错误（面试/开源项目有用）。
推荐：保留！对任何代码都有帮助，不会冲突。

2. Error Lens (usernamehw.errorlens)
作用：内联显示错误/警告信息（把 diagnostic 消息直接显示在代码行旁边，而不是只在 Problems 面板）。
特点：通用插件，提升任何语言的错误可见性（红/黄波浪线 + 文字说明）。
和 Solidity 关系：超级实用！Solidity 编译错误多（pragma 版本、import 路径、gas 等），Error Lens 让问题一眼可见。
推荐：强烈保留！几乎所有 Solidity 开发者都装这个。

3. Even Better TOML (tamasfe.even-better-toml)
作用：TOML 文件（foundry.toml、remappings 等）的语法高亮、补全、格式化、错误检查。
特点：Foundry 项目必备，因为 foundry.toml 是配置文件。
和 Solidity 关系：间接支持 Solidity 开发（配置 compiler 版本、remappings、optimizer 等）。
推荐：保留！如果你用 Foundry，装这个等于优化了配置体验。

4. Solidity (nomicfoundation.hardhat-solidity) → Nomic Foundation 的 Solidity作用：Hardhat 团队（Nomic Foundation）出的官方 Solidity 支持插件。
特点：语法高亮、跳转定义、rename、重构、quick fixes、内联 solc 警告/错误。
自动识别 remappings（Foundry lib/ 依赖如 OpenZeppelin）。
支持 Foundry 项目极好（尽管名字带 Hardhat，但社区公认 Foundry 用这个更友好）。
2024-2026 年主流推荐：Cyfrin Updraft、Reddit、Foundry 文档都推这个。

缺点：Formatter 默认不是 forge fmt（需配置为 Prettier 或其他），但补全/诊断更现代、稳定。
推荐：核心插件，优先保留/设为默认！大多数 Foundry 开发者现在都用这个 + Error Lens。
