看见一些陌生的依赖，比如 dotenv、ethers、react、react-dom 等，这些都是前端项目中常用的依赖。

## dotenv
dotenv 是一个 Node.js 模块，用于从 .env 文件加载环境变量。

```
import dotenv from "dotenv";

dotenv.config();
```

## ethers
ethers 是一个用于与以太坊区块链交互的 JavaScript 库。

## react
react 是一个用于构建用户界面的 JavaScript 库。

## react-dom
react-dom 是 React 库的 DOM 绑定，用于在浏览器中渲染 React 组件。

## next
next 个完整的React 框架，专门用来构建高性能、可扩展、生产级的 Web 应用。它在纯 React 的基础上，增加了大量开箱即用的能力，让你不用自己配置打包工具、路由、数据获取、缓存、SEO 等一大堆东西。

```
"scripts": {
  "dev": "next dev --turbopack",     // 用 Turbopack 启动开发服务器（更快）
  "build": "next build",             // 生产构建（用 Turbopack 默认）
  "start": "next start",             // 启动生产服务器（node server）
  "lint": "next lint"                // 运行内置 ESLint（Next.js 推荐规则）
}
```
next dev --turbopack：2026 年新项目默认推荐，开发体验飞起（启动 < 几秒，改动毫秒刷新）
next build：打包生产版本，生成 .next 目录（静态文件 + 服务端代码）
next start：运行构建后的应用（适合自托管或 Docker）
next lint：检查代码风格（集成 ESLint + Next.js 插件）
